API Estudio crédito de vehículo

Descripción

Capacidad que permite el registro de la información para realizar el estudio preliminar de crédito para véhiculo. Los pasos a seguir en este proceso son:

  • La solución se debe autenticar en la plataforma de APIs de Bancolombia.
  • La solución debe presentar los formularios para la captura de la información para realizar las validaciones  Pre-Bureau,  Bureau y las que apliquen para realizar el estudio preliminar al solicitante del crédito de vehículos, lo cuales estan definidas en los siguientes requerimientos de integración:
    • a. Datos básicos del cliente (RI1-Basic Customer Data) con la cual se capturan los datos basicos del cliente que va adquirir el crédito de vehículo para  realizar el estudio preliminar, donde se genera un identificador único con el cual se enlazará todos los datos a registrar con las siguientes interfaces definidas.
    d. Datos del vehículo a comprar (RI2-Vehicle Information) con la cual se capturan los datos generarles del vehículo a adquirir y la categorización del mismo.
    e. Datos del comercio donde se adquiere el vehículo (RI3-Commerce Information) con la cual se capturan los datos del comercio si aplica o los valores por defecto que se asignarán según el canal.
    b. Datos correspondientes a la actividad económica. (RI4-Economic Activity Customer) con la que se capturan datos principales de la actividad economica del cliente que va adquirir el crédito de vehículo.
  • Una vez realizado este registro de datos preliminares, se entrega una respuesta de si es válido o no continuar con el estudio del crédito, basado en el proceso de validacion de Pre-bureau y Bureau, donde para el backend, ya se contaría con un numero de radicación para el estudio del crédito, invocando los requerimientos de integración de la familia de Estudio Final (Vehicle_Credit_Final_Study)
